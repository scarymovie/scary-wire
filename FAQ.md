# Часто задаваемые вопросы (FAQ)

## Общие вопросы

### Что такое WireGuard?
WireGuard - это современный, быстрый и безопасный VPN протокол. Он использует современную криптографию и работает значительно быстрее, чем OpenVPN или IPSec.

### Какие операционные системы поддерживаются?
**Сервер:**
- Ubuntu 20.04+
- Debian 10+
- CentOS 8+
- RHEL 8+

**Клиенты:**
- Linux (все дистрибутивы)
- Windows 10/11
- macOS
- Android
- iOS

### Безопасен ли WireGuard?
Да, WireGuard использует современные криптографические алгоритмы:
- ChaCha20 для шифрования
- Poly1305 для аутентификации
- Curve25519 для обмена ключами
- BLAKE2s для хеширования

## Установка и настройка

### Сколько времени занимает установка?
Полная установка занимает 2-5 минут в зависимости от скорости интернета и производительности сервера.

### Нужен ли статический IP адрес?
Желательно, но не обязательно. Если IP меняется, используйте скрипт `update-endpoint.sh` для обновления конфигураций клиентов.

### Можно ли изменить порт?
Да, отредактируйте переменную `SERVER_PORT` в скрипте `setup-wireguard.sh` перед установкой.

### Как добавить больше клиентов?
```bash
sudo bash add-client.sh имя_клиента 10.0.0.X
```
Где X - свободный IP адрес от 2 до 254.

## Подключение клиентов

### Как подключиться с мобильного устройства?
1. Установите приложение WireGuard
2. Запустите: `sudo bash show-qr.sh имя_клиента`
3. Отсканируйте QR код в приложении

### Как подключиться с компьютера?
Скопируйте файл конфигурации `/etc/wireguard/имя_клиента.conf` на клиентский компьютер и импортируйте его в WireGuard клиент.

### Почему не работает интернет через VPN?
Проверьте:
1. IP forwarding включен: `sysctl net.ipv4.ip_forward`
2. Firewall настроен правильно: `sudo bash check-status.sh`
3. Правильный публичный IP в конфигурации клиента

### Как настроить split-tunnel?
```bash
sudo bash setup-split-tunnel.sh клиент "10.0.0.0/24, 192.168.1.0/24"
```
Это направит через VPN только указанные сети.

## Производительность

### Какая скорость VPN?
WireGuard может достигать скорости до 1 Гбит/с на современном оборудовании. Реальная скорость зависит от:
- Пропускной способности интернета
- Производительности CPU
- Задержки сети

### Сколько клиентов можно подключить?
Теоретически до 253 клиентов (ограничение подсети /24). На практике ограничение - производительность сервера.

### Влияет ли VPN на задержку?
Минимально. WireGuard добавляет обычно менее 1ms задержки.

## Безопасность

### Как часто нужно менять ключи?
WireGuard использует Perfect Forward Secrecy, поэтому регулярная смена ключей не требуется. Меняйте ключи только при компрометации.

### Что делать если ключ скомпрометирован?
1. Удалите клиента: `sudo bash remove-client.sh имя`
2. Создайте нового: `sudo bash add-client.sh имя 10.0.0.X`
3. Распространите новую конфигурацию

### Логируются ли подключения?
По умолчанию WireGuard не ведет подробные логи. Для мониторинга используйте:
```bash
sudo bash monitor.sh
```

## Резервное копирование

### Что включает резервная копия?
- Все ключи (сервера и клиентов)
- Все конфигурационные файлы
- Настройки сервера

### Как часто делать резервные копии?
Рекомендуется делать резервную копию:
- После установки
- После добавления/удаления клиентов
- Перед обновлением системы

### Где хранить резервные копии?
Храните резервные копии в безопасном месте, отдельно от сервера. Файлы содержат приватные ключи!

## Устранение неполадок

### Клиент не может подключиться
1. Проверьте статус сервера: `sudo bash check-status.sh`
2. Убедитесь, что порт 51820/udp открыт
3. Проверьте правильность публичного IP в конфигурации
4. Проверьте логи: `sudo journalctl -u wg-quick@wg0 -f`

### Нет доступа к интернету через VPN
1. Проверьте IP forwarding: `sysctl net.ipv4.ip_forward` (должно быть 1)
2. Проверьте NAT правила: `sudo iptables -t nat -L`
3. Убедитесь, что DNS настроен в конфигурации клиента

### Низкая скорость VPN
1. Проверьте MTU: попробуйте добавить `MTU = 1420` в секцию [Interface]
2. Проверьте загрузку CPU на сервере
3. Проверьте пропускную способность канала

### Ошибка "Cannot find device wg0"
WireGuard не запущен. Запустите:
```bash
sudo systemctl start wg-quick@wg0
```


### Нужно ли пересоздавать конфигурации после обновления?
Нет, конфигурации остаются совместимыми.

## Удаление

### Как полностью удалить WireGuard?
```bash
sudo bash uninstall.sh
```

Это удалит все конфигурации и создаст резервную копию.

### Сохраняются ли данные после удаления?
Скрипт создает резервную копию перед удалением. Для полного удаления пакетов:
```bash
# Ubuntu/Debian
sudo apt remove --purge wireguard wireguard-tools

# CentOS/RHEL
sudo yum remove wireguard-tools kmod-wireguard
```
