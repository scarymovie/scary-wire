# Настройка обхода .ru и .рф доменов

## Описание

Этот скрипт настраивает split-tunneling с автоматическим обходом российских доменов (.ru, .рф).

## Как это работает

- **Весь трафик** идет через VPN
- **Кроме** трафика на российские IP адреса
- Российские сайты (yandex.ru, vk.com, mail.ru и т.д.) идут **напрямую**
- Зарубежные сайты (google.com, youtube.com и т.д.) идут **через VPN**

## Установка

### Автоматически при установке

При запуске `setup-wireguard.sh` вам будет предложено настроить обход:

```bash
sudo bash setup-wireguard.sh
# Ответьте "y" на вопрос про обход .ru доменов
```

### Вручную после установки

```bash
sudo bash setup-ru-bypass.sh
```

### Через интерактивное меню

```bash
sudo bash quick-start.sh
# Выберите пункт "13) Настроить обход .ru и .рф доменов"
```

## Что делает скрипт

1. Загружает список всех российских IP подсетей (~500+ подсетей)
2. Создает скрипт управления маршрутами
3. Обновляет конфигурации всех клиентов
4. Добавляет PostUp/PostDown хуки для автоматического управления маршрутами

## Проверка работы

### На клиенте после подключения к VPN:

**Проверка российского сайта:**
```bash
curl ifconfig.me
# Должен показать ваш реальный IP (не VPS)

# Или зайдите на yandex.ru и проверьте IP
```

**Проверка зарубежного сайта:**
```bash
curl -4 icanhazip.com
# Должен показать IP вашего VPS

# Или зайдите на google.com и проверьте IP
```

## Технические детали

### Список IP подсетей

Скрипт использует данные из RIPE (Regional Internet Registry) для получения актуального списка российских IP адресов. Если загрузка не удается, используется резервный список основных подсетей.

### Файлы

- `/etc/wireguard/bypass/ru-networks.txt` - список российских подсетей
- `/etc/wireguard/bypass/exclude-ru-routes.sh` - скрипт управления маршрутами

### Как работают маршруты

При подключении VPN:
1. Весь трафик направляется через VPN (0.0.0.0/0)
2. Для каждой российской подсети добавляется маршрут через обычный шлюз
3. Российский трафик идет в обход VPN

При отключении VPN:
1. Все специальные маршруты удаляются
2. Трафик идет как обычно

## Обновление конфигураций клиентов

После запуска скрипта нужно:

1. **Переимпортировать конфигурации** на всех клиентах
2. Или **отсканировать новые QR коды**:
   ```bash
   sudo bash show-qr.sh client1
   ```

## Отключение обхода

Если хотите вернуть обычный режим (весь трафик через VPN):

```bash
# Для конкретного клиента
sudo bash setup-split-tunnel.sh client1 "0.0.0.0/0, ::/0"
```

## Примеры использования

### Пример 1: Обход только российских сайтов
```bash
sudo bash setup-ru-bypass.sh
```

### Пример 2: Обход российских + локальная сеть
Отредактируйте `/etc/wireguard/bypass/ru-networks.txt` и добавьте:
```
192.168.0.0/16
10.0.0.0/8
```

### Пример 3: Проверка маршрутов
```bash
# На клиенте после подключения
ip route | grep -E "185\.|188\.|91\."
# Должны быть маршруты через обычный шлюз
```

## Часто задаваемые вопросы

### Будет ли работать с мобильными приложениями?

Да, работает со всеми приложениями. Например:
- Telegram российские серверы - напрямую
- YouTube - через VPN
- ВКонтакте - напрямую
- Instagram - через VPN

### Как часто обновлять список IP?

Рекомендуется обновлять раз в месяц:
```bash
sudo bash setup-ru-bypass.sh
```

### Можно ли добавить другие страны?

Да, можно добавить любые IP подсети в файл `/etc/wireguard/bypass/ru-networks.txt`

### Влияет ли на скорость?

Нет, маршрутизация происходит на уровне ядра Linux, overhead минимальный.

## Устранение проблем

### Российские сайты все равно идут через VPN

1. Проверьте маршруты:
   ```bash
   ip route | grep 185.
   ```

2. Проверьте скрипт:
   ```bash
   cat /etc/wireguard/bypass/exclude-ru-routes.sh
   ```

3. Переподключите VPN

### Не работает после обновления

Переимпортируйте конфигурацию клиента:
```bash
sudo bash show-qr.sh client1
```

### Хочу вернуть обычный режим

Удалите PostUp/PostDown из конфигурации клиента и переимпортируйте.
